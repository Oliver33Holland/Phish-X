Detect AI-generated phishing

Paste the full raw source of a suspicious email. The raw source reveals hidden routing information, sender authentication results, and domain data that are invisible in the normal email view - giving far more accurate detection than subject and body text alone.

📄 How to copy the raw email source

The raw source contains the full Received chain (server hops), SPF / DKIM / DMARC authentication results, the original sender IP, and the true sending domain - all hidden from the normal email view. This is the same information IT security teams use to investigate suspicious emails.

📷 Outlook / Outlook.com

Open the email → click the ⋯ three-dot menu (top-right) → View > View message source

📷 Gmail

Open the email → click the ⋮ three-dot menu (top-right) → Show original → Copy to clipboard

🍎 Apple Mail

View menu → Message → Raw Source  ⌥⌘U

⛈ Thunderbird

View → Message Source  Ctrl+U

⚠️ Select all the text in the source window (Ctrl+A / Cmd+A) and paste it in the box below.

Include everything from the first Received: line to the end of the message body.

How attackers use fake login pages

Attackers clone real login pages and host them on lookalike domains. They trick victims via phishing emails or SMS into visiting the fake page and entering credentials. Key red flags to teach users:

  • The URL domain doesn't match the real service (always check the address bar).
  • The URL may contain the real brand name but with extra text (e.g. google.com.verify-now.net).
  • The page may look pixel-perfect but the domain is wrong.
  • Real services never ask for your password on unexpected pages.
